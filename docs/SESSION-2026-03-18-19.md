## Tile Preview Node (March 18)
### What it does
Visual seamless tiling checker — connect an image, open a modal viewer to see it tiled in a grid (2×2 to 5×5) with zoom/pan.

### Iterations
1. **First attempt:** Rendered tiles as nested fabric.js groups inside the canvas node → broken, `addWithUpdate` destroyed canvas selection
2. **Second attempt:** Rendered to offscreen `<canvas>`, added as single `fabric.Image` → preview still didn't render correctly
3. **Final approach:** Compact on-canvas node (matching inpaint style) + full-screen modal viewer
   - Node is small (160×60), shows connection status
   - "Open Tile Viewer" button opens modal with the tiled preview
   - Modal has grid size buttons (2×2 to 5×5), zoom (scroll wheel + buttons), pan (click+drag)

### Key lessons
- **Never use `addWithUpdate`/`removeWithUpdate`** on fabric.js groups — it recalculates bounds and breaks selection of all other canvas objects
- **Connection pattern:** Use `data-connect` attributes + `bindProperties()` event listeners (matching inpaint node), not inline `onclick` handlers
- **Disconnect pattern:** Use `data-disconnect` attribute + `removeConnectionBetween()` from engine

## Multi-Image Nano Banana (March 18)
- Both **Nano Banana** and **Nano Banana Pro** now support up to 3 reference images
- Created **`batch_group` system** in `buildWorkflow`:
  - Config inputs declare `batch_group`, `batch_target_node`, `batch_target_field`
  - buildWorkflow auto-inserts `ImageBatch` nodes to chain connected images
  - 1 image → wires directly (no unnecessary batch node)
  - 2–3 images → chains ImageBatch nodes
  - 0 images → text-only mode (optional images deleted from workflow)

## Tripo 3.0 Workflow (March 19)
### Template: `tripo3-img2model`
- Uses `TripoImageToModelNode` (built-in ComfyUI node, NOT the custom `ComfyUI-Tripo` plugin)
- Charges ComfyUI tokens (not separate Tripo API key)
- 3 image inputs (front/side/back) using batch_group system
- Image 1 required, 2 and 3 optional
- Params: model version, category, texture/geometry quality, alignment, orientation, PBR, face limit, seeds
- Includes `Preview3D` node for output

### Debugging journey
1. **No output:** Workflow submitted but `outputs: {}` — turned out to be `execution_cached` from previous run
2. **Missing Preview3D:** Node 5 wasn't in submitted workflow because user's canvas had old cached template without it
3. **Fix:** Made `buildWorkflow` async — now **always fetches latest template from server** before building. Stale templates are never used again.
4. **3D output detection:** Added checks for `text[]` (STRING outputs) and `gltf`/`glb` output keys, not just `result[]`

### 3D Model Saving
- New `/api/comfy/save-mesh` endpoint
- After generation, GLB is fetched from ComfyUI and saved to:
  - `uploads/` (always)
  - `OUTPUT_DIR/3D/` (if configured)
- Renamed with timestamp: `outputName_2026-03-19T13-45-00.glb`
- Sidecar metadata JSON saved alongside
- 3D viewer errors caught gracefully (model still saved even if viewer fails)
